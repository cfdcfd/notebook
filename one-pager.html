Harness Engineering

让 AI 更靠谱地完成复杂任务 · 一页速览

提升推理效果的所有方法 = 监督者的一个动作

可单独分享 · 完整论证见 presentation.html

这些方法不是"功能开关"，而是监督者看情况使用的动作。
就像汽车的辅助驾驶：它不会一直自动刹车、也不会一直保持车道，而是看到什么情况触发什么功能。 我们让一个"监督者"在 AI 干活时全程盯着，按实际情况决定用哪个方法——而不是事先拍板。
事前拍板 = 靠猜 · 运行中观察 = 按真实情况出手 · 同一份运行记录还能反过来调整这张对照表
反直觉：这不是"用更多钱换质量"，而是往往更省钱。 Anthropic 官方测试：Sonnet + Opus 顾问 vs Sonnet 单干，SWE-bench Multilingual 分数 +2.7 pp，同时成本 -11.9%——早介入抵消了后期的失败重试与超长生成。
执行优先级 · MVP 先做已被官方 benchmark 过的，再做我们的增量
① P0.5 · 首发 已验证
复刻 Anthropic Advisor Strategy（Pull 路径）
给 executor（做活的模型）注册一个"求助"工具，它自己判断"这步我拿不准"时主动调用 → 由更强的顾问（Opus / MiniMax / Claude）返回一段 plan，它继续干。

▸ 一次 API 调用内完成，工程改动极小（模仿 advisor_20260301 工具声明 + max_uses 封顶）
▸ 顾问硬边界：不调工具 · 不出用户可见内容 · 只返 plan/correction/stop
▸ 官方数据直接可引用：SWE-bench +2.7 pp · 成本 -11.9% · Haiku 翻倍
② P1 · 增量 我们的差异化
Push 兜底 + 止损规则（我们的增量）
Pull 路径依赖 executor 自己发现问题——但 LLM 常常"自信地错"毫无察觉。我们补两条 Anthropic 方案没有的硬防线：

▸ Push 主动介入：Proxy 看到 logprob 骤降 / 原地打转 / 工具连续失败 → 直接打断
▸ 止损硬规则：同一错误小修 ≥ 2 次 → 强制换策略，固化">2 轮自纠有害"研究结论
▸ 前提：P0 就要开始采集 logprobs / 推理流 / 完整 header（这是 P1 能启动的底座）
为什么这个顺序：先复刻已被验证的方案拿到"提质 + 降本"基本盘（低风险、可对标），再做我们的 push 增量填补 pull 兜不住的死角（差异化价值）。不是两步做成一个大版本，而是 P0.5 就能独立上线看效果。
CORE

监督者总线

一个贯穿做计划 → 写代码 → 查验收全流程的"观察员"。每次 AI 调用模型时它都在一旁盯着，一发现不对劲就出手。

P0.5 先上被请教（pull）
模型自己意识到"这步我拿不准" → 主动调 advisor tool。对应 Anthropic advisor strategy，官方已验证。
P1 增量主动打断（push）
模型"自信地错"没察觉时，监督者根据外部信号强行介入。这是 pull 兜不住的场景，我们补上。

下方表格列的"看到什么→做什么"对两种触发路径都适用；区别在于由谁发起。

监督者看到的情况 监督者做什么 已有研究数据
模型在关键位置（API 名 / 路径 / 数值）突然开始犹豫（置信度骤降）🔑 暂停生成 → 先去查权威文档 → 拿到资料再继续 胡编 API 导致的返工 ↓60-80%
模型的思考时间快用完了，却还没得出结论 插入一句"Wait, 让我再想想"，强迫它多思考一会 +7 个百分点
(s1 论文 AIME24：50%→57%)
模型原地打转 / 长时间没进展 让它换角度想："假设现在方案已经失败，列出 3 个最可能的原因" 无效重试 ↓45%
工具（如查数据库、跑测试）连续失败 ≥ 2 次 把相关文档喂给它 → 再重试；仍不行就换思路 领域知识类幻觉 ↓40-60%
任务属于简单约束类（比如：遵循指令格式、分类、信息抽取） 直接关掉"深度思考"模式
(反常识但有研究支持)		 避免 -10~36% 的反向退化
⚠ 同一个错误已经"小修过 2 次"仍没修好 强制换策略（见下方档位） > 2 轮自我纠错反而会把模型带偏
换策略档位（五选一）：
①让模型独立想 N 次，选最优的那个 ②让模型独立想 N 次，合并各版本的好处（比单选最优再 +5~8%） ③强制先查资料再重做 ④请更强的模型当顾问（局部上场，不必全程用） ⑤多视角交叉评审 / 转人工
档位 ④ 的依据：Anthropic 实测 Haiku 4.5 在 BrowseComp 上 19.7% → 加 Opus 顾问后 41.2%（翻倍），比 Sonnet 单跑便宜 85%。大多时候不必把全程换成大模型，只在关键决策点请教一下就够。
P1 增量止损规则（硬规则）：同一个问题上，用"小修"（续思、重注入提示、唱反调、让它先复述问题）累计修了 2 次还没修好，就禁止继续小修，必须切换到上面的"换策略档位"。这条规则把研究结论"超过 2 轮自我纠错反而有害"固化成强制协议，避免 AI 在错误方向上越陷越深。
说明：P0.5 先把 pull 路径（advisor tool）上线拿到基本盘；止损规则和 push 主动介入作为 P1 增量，兜住 executor "自信地错"没主动求助的场景——这是 Anthropic advisor strategy 之上我们的差异化防线。
